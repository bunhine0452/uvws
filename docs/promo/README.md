## 올릴 때 팁

- **한 번에 다 뿌리지 말기.** 며칠 간격으로 하나씩 올리고 반응 보면서 글 다듬기. 같은 날 도배하면 역효과.
- **핏 좋은 채널부터.** ComfyUI/StableDiffusion 서브레딧이 가장 잘 맞습니다(현재 본인이 겪는 고통 그대로). 단, **서브별 자기홍보 규칙을 반드시 먼저 확인** — 밴 한 번이면 채널 하나를 잃습니다.
- **데모 GIF > 글.** preview.png는 기본, 짧은 GIF/영상 하나가 전환율을 가릅니다.
- **첫 댓글이 중요.** HN / Product Hunt는 "왜 만들었나"를 메이커가 직접 적으면 반응이 훨씬 좋아요(각 파일에 코멘트 초안 포함).
- **질문엔 빨리 답하기.** 올린 직후 1~2시간이 트래픽 피크라 댓글 대응이 곧 다운로드.
- **솔직하게.** 사이드 프로젝트·미공증·CPU/RAM(VRAM 아님) 같은 한계는 숨기지 말 것. 개발자 커뮤니티는 오히려 그런 데서 신뢰합니다.
